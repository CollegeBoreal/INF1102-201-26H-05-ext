Qu’est‑ce qu’Ansible ?

Ansible est un outil d’Infrastructure as Code (IaC) orienté gestion de configuration. Il permet d’automatiser l’installation de logiciels, la configuration de services comme Nginx, et le déploiement de fichiers de manière idempotente et reproductible.

Concepts IaC / Ansible

  • Approche déclarative → On décrit l’état final, pas les étapes.
  • Playbook YAML → Tâches ordonnées pour configurer les serveurs.
  • Impératif vs déclaratif → Bash/Python = étapes ; Ansible = état final.
  • Handlers & notifications → Redémarrer un service seulement si nécessaire.

Ce que fait mon playbook

  • Installer nginx sur la VM via le module apt.
  • Copier mes fichiers index.html, style.css et script.js sur le serveur.
  • Démarrer le service nginx et l’activer au boot.

Ansible dans l’écosystème IaC

Ansible se positionne dans la partie configuration d’un pipeline DevOps :
Terraform / Tofu → provisionning de l’infrastructure (VM, réseau…)
Ansible → configuration des machines (nginx, utilitaires, sécurité…)
